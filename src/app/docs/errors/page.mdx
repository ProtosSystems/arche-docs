# Errors

All errors returned by the Arche API follow a standard structure.

## Error Envelope

`error.code`: `VALIDATION_ERROR`  
`error.http_status`: `400`  
`error.message`: `from must be <= to`  
`error.trace_id`: `7e8a5d2e`

## Error Fields

- `code`: machine readable error code
- `http_status`: HTTP response code
- `message`: human readable message
- `trace_id`: request identifier for debugging

## Common Errors

- `VALIDATION_ERROR`: request parameters invalid
- `NOT_FOUND`: resource does not exist
- `RATE_LIMITED`: too many requests
- `INTERNAL_ERROR`: unexpected server failure
