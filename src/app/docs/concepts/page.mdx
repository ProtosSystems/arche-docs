# Core Concepts

Understanding Arche requires understanding how financial data evolves over time.

## Financial Statements Are Revised

Corporate financial statements can change after publication.

Common reasons include:

- accounting corrections
- amended filings
- regulatory adjustments
- classification changes

Most vendors overwrite historical values.

Arche preserves each version.

## Statement Versions

Each statement has a version history.

| Version | Filing         | Accepted At |
| ------- | -------------- | ----------- |
| 1       | Original 10-K  | Nov 3 2023  |
| 2       | Amended filing | Feb 12 2024 |

Queries can retrieve:

- the latest version
- the version known at a specific date
- the full version history

## Point-in-Time Queries

A **point-in-time query** returns the statement version that was available at the specified time.

```http
GET /statements?as_of=2024-01-01
```

This ensures historical research uses only information available at the decision date.

## Deterministic Datasets

A dataset is deterministic when:

- the same query always returns the same result
- historical values do not drift over time

Arche guarantees deterministic queries through version preservation.
