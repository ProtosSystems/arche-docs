# Company Resolution and Statement Retrieval

This is the basic operational workflow for moving from a ticker to actual Arche statement data.

## Resolve the Company

```http
GET /v1/edgar/companies:resolve?ticker=AAPL
```

Use the resolved company record to obtain the canonical `cik`.

## Inspect Filings

Once you have a `cik`, you can inspect filing history.

```http
GET /v1/edgar/companies/{cik}/filings
```

If you need a single filing:

```http
GET /v1/edgar/companies/{cik}/filings/{accession_id}
```

## Retrieve Statements

List company statements directly:

```http
GET /v1/edgar/companies/{cik}/statements?statement_type=INCOME_STATEMENT&page=1&page_size=10
```

Or inspect statements within a specific filing:

```http
GET /v1/edgar/companies/{cik}/filings/{accession_id}/statements
```

This is the route split to keep in mind:

- filing routes answer document-history questions
- statement routes answer normalized-financial-data questions

## Move into Point-in-Time Analysis

Once you know the issuer and the statement family you care about, switch to point-in-time retrieval when the historical boundary matters.

```http
GET /v1/edgar/as-of/{date}/financials/{cik}
```
