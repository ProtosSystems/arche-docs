## Start Here

If your use case depends on historical correctness, Arche is usually about one of four tasks:

- resolve a company identifier into a canonical issuer record
- retrieve statements or filings for that issuer
- inspect statement version lineage across restatements
- query what was knowable on a specific date

## What Arche Solves

Traditional datasets often expose the latest corrected history. That is convenient for present-day analysis, but it breaks reproducibility when historical values change later.

Arche preserves:

- versioned statement lineage
- point-in-time query semantics
- deterministic retrieval for the same request

<Warning>
  <p>
    If you backtest with the latest restated values instead of what was actually knowable on the
    decision date, your historical results can drift over time.
  </p>
</Warning>

## First Request

Resolve a ticker into a canonical issuer record:

<CodeGroup title="Resolve a company">
```bash
curl -X GET "https://api.arche.fi/v1/edgar/companies:resolve?ticker=MSFT" \
  -H "X-Api-Key: YOUR_API_KEY" \
  -H "Accept: application/json"
```

```ts
const response = await fetch(
  'https://api.arche.fi/v1/edgar/companies:resolve?ticker=MSFT',
  {
    headers: {
      'X-Api-Key': process.env.ARCHE_API_KEY!,
      Accept: 'application/json',
    },
  },
)

const payload = await response.json()
console.log(payload.data)
```

```python
import os
import requests

response = requests.get(
    "https://api.arche.fi/v1/edgar/companies:resolve",
    params={"ticker": "MSFT"},
    headers={
        "X-Api-Key": os.environ["ARCHE_API_KEY"],
        "Accept": "application/json",
    },
    timeout=30,
)

print(response.json()["data"])
```
</CodeGroup>

Representative response:

```json
{
  "data": {
    "as_of": "2026-03-01",
    "cik": "0000789019",
    "exchange": "NASDAQ",
    "ticker": "MSFT"
  }
}
```
